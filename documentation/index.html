Celebration Cards iOS App

This complete app written in Swift 2.0 brings to the users a celebration cards creator where they can customize the available templates by setting a message and changing the main image.

The app is bundled with lots of templates and categories ranging from Christmas to Love or Birthday. Once the user has customized it he/she can share it to any app installed in his device as Facebook, Twitter or WhatsApp.

Moreover you can almost directly built and make revenue out of this app as it also includes Google AdMob banners and interstitials support.

Features

This app has been developed using Swift 2.0 and Xcode in order to provide a ready to use and customizable template to be integrated in any app or to be released alone itself.

 

Here are some of the most important features:

  • Create Celebration Cards: Easily, choose from dozens of templates and customize them to make your card.
  • iPhone & iPad support: Works on any model of this family of devices.
  • iOS8 & iOS9: Specially coded to work on the last OS major updates.
  • Flat design: It has a flat design matching current web and app design trends.â€¨
  • Easy customization: It has a minimalist touch and so there only a few elements. Concerning this fact and the use of the Storyboard, it is easy to change colors and fonts.
  • Use of Storyboard: In order to provide a visual reference and speed up development.
  • Clean & commented code: The code tries to be as basic as possible and there are comments on the most important parts.
  • Auto resize with all devices: The UI adapts itself using NSConstraints and shows or hides certain elements to fit all screens.
  • Share to any app: Thanks to in-built share protocol, you can share your final image using any app installed on your device.
  • Google AdMob support: We added a bottom banner as well as a interstitial view to display Google AdMob ads so you can make some revenue out of this app.

Content Structure

There is the typical structure of an Xcode iOS Swift 2.0 project:

  • [folder] Main Folder
    • GoogleMobileAds.framework
    • Other frameworks...
  • [folder] CelebrationCards
    • AppDelegate.swift
    • MenuViewController.swift
    • CategoriesViewController.swift
    • TemplatesViewController.swift
    • CardViewController.swift
    • Main.storyboard
    • LaunchScreen.storyboard
  • [folder] Resources
    • [folder] Images
      • logo.png
      • back.png
      • back_white.png
      • share.png
      • camera_icon.png
      • [folder] Card Decoration
        • All resource images used to create the templates

 

The Main folder contains all the frameworks needed to display AdMob ads.

The CelebrationCards folder contains the main files of the app such as the AppDelegate.swift file which is used to init the app but it's the default version as it has not been modified, the MenuViewController.swift file which works as the cover or intro screen, the CategoriesViewController.swift file which is used to display all the available categories, the TemplatesViewController.swift file which is used to show the available templates, the CardViewController.swift used to display and create the final card and the Main.storyboard file which is used to create and style the structure and the UI elements.

The Images folder inside the Resources folder contains the images the app uses. If you want to change the look and feel of the app you can substitute your new images for these ones. Also, there is another folder, the Card Decoration, that contains the resources used to create cards.

Quick Start Guide

The app is very simple and in a few steps can be fully customized:

 

  • Change the menu screen:

The menu screen consists on two elements: the logo and the start button.:

  • To change the logo it is as easy as import a new photo named logo.png and drop it in the Resources -> images folder.


  • For the start button as a native button, you can change its background color, font family, size and color as well as its corner radius:




  • Change the categories screen:

The categories screen consists of a navigation bar, the collection view and the ads view. From this elements you can customize some properties to make it more your way:

  • To change the nav bar it is as easy as go to the storyboard and change some properties as the tint color.


  • For the button on the nav bar, as usual, you can change it by loading the image and set them as the button image. You have to load your new image (29x22px, 57x44px and 86x66px) and name it back_white.png, back_white@2x.png and back_white@3.png.


  • Concerning the categories collection view you can edit its background color as well as the corner radius and size of each cell. The background color and the corner radius can be changed in the Storyboard while the size of each cell is defined in CategoriesViewController at line 78.






  • Add more categories:

Right now, the app is bundled with 5 categories. However if you would like to add more categories, here is how it's done:

  • First, you should load the images that represents each new category by adding them to Card Decoration folder.


  • Then you should go to CategoriesViewControler at line 18 and add the name of those images to the categories array:


  • Also, of course, you should add templates to this category in the TemplatesViewController as you can see here:.


  • Change the templates screen:

The templates screen is similar to the categories screen as also consists of a navigation bar, the collection view and the ads view. The only difference is the collection view cells distribution. As well, from this elements you can customize some properties to make it more your way:

  • To change the nav bar it is as easy as go to the storyboard and change some properties as the tint color.


  • For the button on the nav bar, as usual, you can change it by loading the image and set them as the button image. You have to load your new image (29x22px, 57x44px and 86x66px) and name it back_white.png, back_white@2x.png and back_white@3.png.


  • Concerning the categories collection view you can edit its background color as well as the shadow and size of each cell. The background color can be changed in the Storyboard while the shadow of each cell is defined in TemplatesViewController at line 133 and the size of each cell is defined in TemplatesViewController at line 145. It should be noted that the app uses two different cells to fit the landscapes and portraits templates.






  • Add more templates:

Right now, the app is bundled with more than 40 templates. However if you would like to add more templates, here is how it's done:

  • First, you should load the images that represents each new template and the ones that are used to build them by adding them to Card Decoration folder.


  • Then you should go to TemplatesViewControler at line 54 and add the needed information of each template to the templates array. You need to set the background_image for the image that is going to be displayed in this collection view, the type for the template type you want to use it, the text for the placeholder text on the template, the image for the background image on the card, the image2 for the decoration image on the card, the background color for the background color on the card and the foreground color for the text color on the card:


  • Moreover, instead of adding new templates you can create a new template, meaning a new layout. To do so, you have to go to Main.storyboard and create a new view using one of the current templates as a starting point. Then you can move and resize any of the elements to create your new template.


  • Finally, after you created this new templates you need to go to CardViewController at line 142 and add a new case for your new template, set if is a portrait or landscape template as well as create a new variable for it and link it to the interface Builder..




  • Change the card screen:

The cards screen is a little bit different to the rest of screens as also consists of a two top buttons, a scroll view containing the card view and the ads view. You can only customize the top buttons.:

  • To customize the top buttons it is as easy as load two new images named back.png and share.png. Check the existent ones to know which sizes are needed.


  • â€‹How to add ads (Google AdMob or Apple iAD):

  • To enable Google AdMob you need to go to CategoriesViewController.swift, TemplatesViewController.swift and CardViewController and search and change the value of useAds to 1.

  • To change your AdMob id you just need to go to CategoriesViewController.swift, TemplatesViewController.swift and CardViewController and search and change the value of adUnitID to your AdMob id for the banner and interstitialAdUnitID for the interstitial.

  • In case you don't want to use ads you just need to go to CategoriesViewController.swift, TemplatesViewController.swift and CardViewController and search and change the value of useAds to 0.
